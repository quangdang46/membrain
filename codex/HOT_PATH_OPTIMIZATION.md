# HOT PATH OPTIMIZATION

This document records the engineering techniques required to make the latency targets plausible.

## 1. CPU cache locality

### Objective
Optimize cpu cache locality for bounded latency and minimal variance.

### Tactics
- choose stable and compact representations
- avoid dynamic allocations where possible
- split hot metadata from cold payloads
- cap work by query class
- prefer precomputed handles over expensive reconstruction

### Tradeoffs
- more complexity in implementation
- harder debugging
- stricter invariants
- more specialized benchmarking requirements
