# NEURO MAPPING

This document maps human-memory-inspired functions into computational primitives for agents.
The goal is not biological imitation. The goal is functional translation with performance constraints.

## 1. Working memory

### Human-side intuition
Working memory exists in human cognition as a functional capability, not as a literal one-to-one storage mechanic.

### Agent-side translation
For membrain, working memory maps to a combination of storage tier, retrieval mode, background jobs, and policy logic.

### Computational primitives
- event representation
- metadata and salience
- routing decisions
- index support
- graph support
- reinforcement and decay updates

### Main risks
- overfitting biological analogy
- excessive compute in fast path
- ambiguous boundaries between memory types
- accidental duplication of the same concept in multiple tiers

### Engineering note
Every mapping for working memory should be expressed as interfaces, budgets, and quality metrics rather than metaphors.
