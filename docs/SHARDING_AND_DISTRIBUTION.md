# SHARDING AND DISTRIBUTION

This document captures scale-out strategies and their operational implications.

## 1. Namespace sharding

### Strategy
Use namespace sharding only when its access patterns match the dominant workload.

### Pros
- better locality for some workloads
- bounded shard sizes
- easier maintenance windows

### Cons
- cross-shard recall complexity
- rebalancing cost
- metadata coordination overhead
